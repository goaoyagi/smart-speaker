# AIスマートスピーカー 機能拡張バックログ（将来要件）

本ドキュメントは、基本機能（耳・検索・構成・脳・口の5ステップ）の構築完了後に実装する、UX向上および機能拡張のためのロードマップである。

---

## 1. ウェイクワードによる起動（音声トリガー）
- **概要**: 
  ボタンを押して起動するのではなく、常時マイクの入力を監視し、特定のキーワード（例：「オッケー、スピーカー」など）を検知した瞬間に自動でメインの会話ループ（`main.py`）を起動する。
- **考慮される技術要素**:
  - `OpenWakeWord` や `Porcupine` などのローカルで軽量に動作するウェイクワード検出ライブラリの選定。
  - 常時マイク入力をキャッチし、キーワード検知時にシグナルを送るバックグラウンドプロセスの実装。

## 2. ステータスLEDの点灯制御（視覚フィードバック）
- **概要**: 
  スピーカーが現在どのような状態にあるか（ユーザーの話を聴いている最中、Web検索中、LLM思考中、発話中）を、ラズパイのGPIOピンに接続したLEDの点灯・消灯・点滅パターンで視覚的にユーザーに伝える。
- **考慮される技術要素**:
  - `gpiozero` または `RPi.GPIO` ライブラリを使用したラズパイの物理ピン制御。
  - `listener.py`（聞き取り開始時に点灯）や `retriever.py`（検索中に点滅）など、各コンポーネントの状態遷移イベントに連動する非同期制御ロジック。

## 3. ローカルドキュメントの検索とインデックス化（プライベートRAG）
- **概要**: 
  外部のWeb検索（SearXNG）だけでなく、ラズパイ内の特定のフォルダに置かれた私的なファイル（Markdown、テキスト、PDFなど）を自動で読み込んでインデックス化し、ユーザーからの質問に対してそのローカル情報からも回答できるようにする。
- **考慮される技術要素**:
  - `ChromaDB` や `FAISS` などの、ラズパイで軽量に動くローカルベクトルデータベースの導入。
  - 特定フォルダ内のファイルを監視し、更新があれば埋め込み（Embedding）ベクトルに変換してDBに登録するバックグラウンドスクリプト。
  - `retriever.py` を拡張し、Web検索結果とローカルDBの検索結果をマージしてハイブリッドな事実ソースを作るロジック。

## 4. 会話コンテキストの保持（履歴管理・マルチターン対話）
- **概要**:
  「もう一回言って」というリクエストに対して直前の回答を再復唱したり、「それはどういうこと？」といった直前の文脈を踏まえた深掘りの質問に対応できるよう、過去N回分の「問いと答え」のペアを記憶・保持する。
- **考慮される技術要素**:
  - **Sliding Window Memory**: `collections.deque` を使用し、maxlenを設定して自動的に古い履歴を破棄。O(1)のappend/popleftで効率的に管理。
  - **Condense Question**: 過去の対話履歴を要約して現在の質問に統合し、プロンプト長を削減しつつ文脈を保持。例：「ユーザーは以前Xについて質問し、Yと回答された。その後Zについて質問し、Wと回答された。現在の質問は...」
  - 新規モジュール `src/conversation_history.py` を作成し、`ConversationHistory` クラスで履歴管理と要約ロジックを実装。
  - `composer.py` を拡張し、要約された履歴をプロンプトにフォーマットして埋め込み、`brain.py` へ渡す仕組み。
  - `main.py` で対話完了後に履歴を追加し、特殊コマンド（「もう一回言って」など）を検知して履歴を再利用。

## 5. プッシュトゥトーク（ボタン押下中のみ聞き取り）
- **概要**:
  固定秒数（`RECORD_SECONDS`）の録音ではなく、GPIOに接続した物理ボタンを**押している間だけ**マイク入力を録音し、**ボタンを離した瞬間**に録音を停止して文字起こし→検索→生成→発話のパイプラインを開始する（いわゆるプッシュトゥトーク方式）。
- **ユーザー体験**:
  1. 待機中: LEDは消灯（`IDLE`）。
  2. ボタン押下: 録音開始、LEDは点灯（`LISTENING`）。
  3. ボタン離す: 録音停止、以降は既存パイプライン（検索→思考→発話）へ遷移。
- **考慮される技術要素**:
  - **GPIOボタン入力**: `gpiozero.Button` で押下（`when_pressed`）・離す（`when_released`）イベントを検知。チャタリング対策として `bounce_time`（例: 50ms）を設定。
  - **録音方式の変更**: 現行の `arecord -d <秒数>` は固定長録音のため、押下中のみ録音するには `sounddevice` / `PyAudio` などでストリーミング録音し、離したタイミングでバッファを確定する方式へ `listener.py` を拡張する必要がある。
  - **最短・最長録音時間**: 誤タップ防止のため最短（例: 0.3秒）未満は無視。暴走防止のため最長（例: 30秒）で強制停止。
  - **無音判定**: ボタンを離した時点で音声レベルが閾値未満なら、既存と同様に「音声が検出されませんでした」とフィードバックして `IDLE` に戻す。
  - **状態LEDとの連動**: 押下中は `LISTENING`、離した後は既存の `SEARCHING` → `THINKING` → `SPEAKING` → `IDLE` 遷移を維持。
  - **発話中の割り込み（バージイン）**: 回答再生中にボタンが再押下された場合、TTS再生を中断して新たな聞き取りに切り替えるか、押下を無視するかを仕様として決める。
  - **他トリガーとの共存**: ウェイクワード起動（#1）や連続会話ループ（Phase C）と併用する場合、起動モードを環境変数（例: `TRIGGER_MODE=push_to_talk|wake_word|fixed_duration`）で切り替え可能にする。
  - **テスト**: GPIOは `pytest-mock` または `gpiozero` の `MockFactory` でモック化し、押下→離すシーケンスと録音バッファの受け渡しをユニットテストする。
  - **新規モジュール候補**: `src/push_to_talk.py` に `PushToTalkButton` クラスを作成し、ボタンイベント管理と `main.py` のオーケストレーションを分離する。
